# 松境心理微信小程序迁移设计

## 背景

当前项目是一个静态官网，位于 `mytest/`，由 7 个 HTML 页面、一个全局 CSS 文件、一个全局 JS 文件以及图片资源组成。网站内容完整，适合迁移成微信小程序的展示型版本。

第一版小程序目标不是做复杂预约系统，而是把现有官网内容整理成更适合微信内浏览和联系的服务名片：展示品牌、服务、团队、理念与联系方式，让用户能快速联系微信客服、拨打电话或打开地图导航。

## 目标

- 保留现有官网的主要内容和温暖、克制、专业的视觉气质。
- 将网站导航重组为小程序底部 5 个 tab。
- 不做预约表单，不在第一版收集用户姓名、手机号或咨询内容。
- 提供清晰直接的联系入口：微信客服或二维码、电话、地图定位、工作时间。
- 控制小程序包体，压缩咨询师照片和地图图片。
- 对心理咨询相关文案做轻微合规处理，避免医疗诊断、治疗、治愈承诺等表达。

## 非目标

- 不接入在线支付。
- 不做咨询师排班。
- 不做用户登录、用户档案、预约后台或订单系统。
- 不做自动心理测评、AI 咨询或医疗诊断相关功能。
- 不完全逐像素复刻网站桌面布局。

## 信息架构

小程序使用 5 个底部 tab：

1. 首页
2. 服务
3. 团队
4. 关于
5. 联系

### 首页

来源主要为 `index.html`。

首页保留品牌第一印象和核心转化入口，但比网页更短。建议结构：

- 顶部品牌区：松境心理、主标题、简短介绍。
- 快速联系区：微信咨询、拨打电话、地图导航。
- 常见困扰卡片：情绪与压力、关系与家庭、青少年成长、个人探索。
- 服务概览：个人心理咨询、青少年与家庭支持、情感与关系咨询。
- 咨询流程简版：初步沟通、咨询师匹配、正式咨询、持续跟进。
- 团队预览：展示 3 位咨询师，并引导到团队页。

所有“预约咨询”按钮在小程序中改为“微信咨询”或“联系咨询顾问”。

### 服务

来源主要为 `services.html`，并吸收 `about-us.html` 中和咨询特色相关的内容。

服务页按卡片分组：

- 服务介绍：松境咨询的定位和支持方式。
- 咨询方向：心理/情绪困扰、青少年成长、亲密关系、职场、个人成长。
- 咨询阶段：缓解心理压抑、初步化解恐惧、看到深层控制、恢复感知。
- 咨询流程：联系客服顾问、初始评估、选择咨询师、签署咨询协议、咨询、持续关怀与回访。
- 服务形式：咨询、直播课程、成长营/沙龙、长期支持计划、咨询师培养。
- 咨询须知：保密、改期、退款、协议等条款。

咨询须知中的医疗边界保留并加强，明确咨询服务不等同于医疗诊断或治疗。

### 团队

来源主要为 `counselors.html`。

团队页保留创始人和咨询师列表，但改成适合手机浏览的卡片列表：

- 页面简介：团队理念和咨询师定位。
- 创始人区：边志瑜简介。
- 咨询师特质：亲历者、真心助人、咨询是艺术、以人为本。
- 咨询师列表：头像、姓名、身份、擅长方向。

咨询师卡片第一版不做详情页也可以，但推荐做可展开卡片或详情页，避免列表过长。所有咨询师照片需要压缩后放入小程序资源目录。

### 关于

来源主要为 `about-us.html` 和 `achievements.html`。

关于页合并官网中的品牌理念、特色和成果：

- 关于松境：团队介绍和品牌故事。
- 松境理念：心的觉悟、真心相伴、大爱。
- 咨询特色：心理困扰支持、情感家庭问题、青少年问题、自我成长。
- 核心难题：围绕“停不下来”的理解和支持。
- 咨询师特质：可与团队页做轻量重复，但不要完全冗长复制。
- 松境数据、客户评价、合作企业。

客户评价如涉及真实用户，应避免暴露可识别身份信息。

### 联系

来源主要为 `contact.html`。

联系页作为关键转化页，信息直接展示：

- 微信咨询入口：优先使用小程序客服按钮；如尚未开通客服能力，则展示现有二维码 `file/code.png`。
- 电话：`13221940530`，点击后调用 `wx.makePhoneCall`。
- 邮箱：`consult@songjingpsy.com.cn`。
- 地址：`海曙区高桥镇梁祝路12弄良库梧桐里D3-102号`。
- 工作时间：周一至周五 9:00 - 18:00，周六 9:00 - 12:00。
- 地图导航：调用 `wx.openLocation`，使用 `gcj02` 坐标系。

地图经纬度在实施前需要确认。若只提供地址，需要先查到准确坐标。

## 交互设计

### 底部导航

使用微信小程序原生 tabBar：

- 首页
- 服务
- 团队
- 关于
- 联系

tabBar 文案简短，图标可使用简单线性图标或先用无图标版本占位，最终上线前补齐。

### 联系入口

每个主要页面至少有一个明显联系入口：

- 首页 hero 区提供“微信咨询”“拨打电话”“地图导航”。
- 服务页在流程和须知后提供“联系咨询顾问”。
- 团队页在咨询师列表前后提供“联系咨询顾问”。
- 关于页底部提供“了解服务”或“联系松境”。
- 联系页集中展示所有联系方式。

### FAQ

官网 `faq.html` 的内容第一版可以放在服务页下方，或在“关于”页保留一个“常见问题”分组。交互使用折叠面板。

如果内容过长，实施时优先放入服务页，后续再拆成单独页面。

## 技术方案

### 工程类型

使用原生微信小程序工程，而不是 WebView 包网页。原因：

- 小程序审核更清晰。
- 可以直接使用 `wx.makePhoneCall`、`wx.openLocation`、客服按钮等能力。
- 页面加载更轻，体验更像微信内产品。

### 页面文件

建议目录结构：

```text
miniprogram/
  app.js
  app.json
  app.wxss
  pages/
    home/
    services/
    team/
    about/
    contact/
  assets/
    images/
```

### 数据组织

第一版内容可以写在各页面 JS 的本地数据中，避免引入后端。咨询师列表建议抽成单独数据文件，便于维护。

后续如需要后台管理，再迁移到云开发或自有 API。

### 资源处理

当前资源中部分图片较大，尤其咨询师照片和地图预览图。实施时需要：

- 将大图压缩到适合小程序的尺寸。
- 咨询师头像建议裁切为统一比例。
- SVG 插画如能正常使用可保留；如遇小程序兼容问题，转为 PNG。
- 地图预览图可以压缩，或直接用地址卡片替代，减少包体。

## 合规与审核

心理咨询类小程序文案需要保持专业边界：

- 避免“治疗疾病”“诊断”“治愈”“保证效果”等表述。
- 建议使用“心理咨询支持”“情绪压力支持”“理解困扰”“改善生活质量”等表达。
- 明确心理咨询不能替代必要医疗诊断和治疗。
- 对严重危机情况提示用户联系正规医疗机构或紧急救助渠道。
- 不在第一版收集敏感个人信息，隐私压力较低。

上线前需要用户在微信公众平台完成企业主体注册、认证、备案、类目选择和最终审核提交。

## 需要用户提供或确认

- 小程序 AppID。
- 小程序名称。
- 客服方式：小程序客服、企业微信客服，或仅展示二维码。
- 线下地址是否就是：海曙区高桥镇梁祝路12弄良库梧桐里D3-102号。
- 地图经纬度，或允许根据地址查询并确认坐标。
- 咨询师照片和资料是否全部沿用当前网页。
- 是否保留“我们的成果”中的数据、客户评价和合作企业。

## 测试计划

- 微信开发者工具预览检查。
- iPhone 真机检查 tabBar、滚动、图片、客服、电话、地图。
- Android 真机检查 tabBar、滚动、图片、客服、电话、地图。
- 检查包体大小。
- 检查所有按钮是否可用。
- 检查敏感文案和审核说明。

## 推荐第一版范围

第一版实现 5-tab 展示型小程序：

- 首页
- 服务
- 团队
- 关于
- 联系

包含微信咨询入口、电话拨打和地图导航。不包含表单、登录、支付、排班、后台管理。

这个范围足够上线使用，也保留后续扩展空间。
